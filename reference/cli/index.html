    CLI Reference

    This page provides documentation for the command line interface.

    tinymotion-backend

    Command line interface to TinyMotion Backend

    Usage:

    tinymotion-backend [OPTIONS] COMMAND [ARGS]...

    Options:

      --help  Show this message and exit.

    consent

    Commands to interact with consents in the backend.

    Usage:

    tinymotion-backend consent [OPTIONS] COMMAND [ARGS]...

    Options:

      --help  Show this message and exit.

    create

    Create a new consent.

    Usage:

    tinymotion-backend consent create [OPTIONS]

    Options:

      -u, --user-id UUID              ID of the User to create the Consent
                                  [required]
  -i, --infant-id UUID            The infant_id of the infant this Consent
                                  relates to  [required]
  -n, --consent-giver-name TEXT   The Consent giver's name
  -e, --consent-giver-email TEXT  The Consent giver's email
  -p, --collected-physically      Flag to indicate if the Consent was
                                  collected physically
  --help                          Show this message and exit.

    list

    List existing consents.

    Usage:

    tinymotion-backend consent list [OPTIONS]

    Options:

      -p, --pager  Display results using a pager
  --help       Show this message and exit.

    infant

    Commands to interact with infants in the backend.

    Usage:

    tinymotion-backend infant [OPTIONS] COMMAND [ARGS]...

    Options:

      --help  Show this message and exit.

    create

    Create a new infant.

    Usage:

    tinymotion-backend infant create [OPTIONS]

    Options:

      -u, --user-id UUID           ID of the User to create the Infant  [required]
  -f, --full-name TEXT         The infant's full name  [required]
  -n, --nhi-number TEXT        The infant's NHI number  [required]
  -b, --birth-date [%Y-%m-%d]  The infant's date of birth  [required]
  -d, --due-date [%Y-%m-%d]    The infant's expected date of birth  [required]
  --help                       Show this message and exit.

    delete

    Delete the specified Infant.

    INFANT_ID is the id of the infant to delete.

    Usage:

    tinymotion-backend infant delete [OPTIONS] INFANT_ID

    Options:

      --help  Show this message and exit.

    list

    List existing infants.

    Usage:

    tinymotion-backend infant list [OPTIONS]

    Options:

      -p, --pager  Display results using a pager
  --help       Show this message and exit.

    user

    Commands to interact with users in the backend.

    Usage:

    tinymotion-backend user [OPTIONS] COMMAND [ARGS]...

    Options:

      --help  Show this message and exit.

    create

    Create a new user.

    Usage:

    tinymotion-backend user create [OPTIONS]

    Options:

      -e, --email TEXT       The new user's email  [required]
  -k, --access-key TEXT  The new user's access key that they will use to login
                         [required]
  -d, --disabled         Disable the user's account on creation
  --help                 Show this message and exit.

    delete

    Delete the specified User.

    USER_ID is the id of the user to delete.

    Usage:

    tinymotion-backend user delete [OPTIONS] USER_ID

    Options:

      --help  Show this message and exit.

    list

    List existing users.

    Usage:

    tinymotion-backend user list [OPTIONS]

    Options:

      -p, --pager  Display results using a pager
  --help       Show this message and exit.

    update

    Update the specified User.

    USER_ID is the id of the user to update.

    Usage:

    tinymotion-backend user update [OPTIONS] USER_ID

    Options:

      -e, --email TEXT        Update the user's email
  -k, --access-key TEXT   Update the user's access key
  -d, --disabled BOOLEAN  Update whether the user is disabled
  --help                  Show this message and exit.

    version

    Print the version of the TinyMotion backend and exit.

    Usage:

    tinymotion-backend version [OPTIONS]

    Options:

      --help  Show this message and exit.
